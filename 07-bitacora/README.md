### Conceptos que te costaron entender

```
Al principio no entendía la diferencia entre return y print.
print() muestra algo en pantalla, pero la función no devuelve nada.
return devuelve un valor que puedes guardar en una variable.
```

### Trucos que descubriste

```
Para saber el tipo de una variable: type(variable)
Para ver el contenido de una lista: print(lista)
Para debuggear: poner print() antes y después del código problemático
```

## Consejos

1. **Escribe mientras aprendes**, no después. La memoria engaña.
2. **Sé honesta.** Si algo no tiene sentido, escríbelo. Es información valiosa.
3. **Copia errores completos.** Más tarde te servirán para entender qué pasó.
4. **Registra victorias.** Cuando algo funcione después de mucho intentar, anótalo.

## Un último recordatorio

Aprender a programar es frustrante a veces. Vas a cometer errores tontos. Vas a olvidar poner dos puntos después de un `if`. Vas a confundir `=` con `==`.

**Todo eso es completamente normal.** Todas las personas que programan pasan por eso. No eres más lenta ni menos capaz. Estás aprendiendo algo nuevo y eso requiere tiempo.

Tu bitácora es el testimonio de ese viaje. Dentro de unas semanas la releerás y verás cuánto avanzaste.
